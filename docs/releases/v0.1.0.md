# v0.1.0 Release Notes

`v0.1.0` 是当前仓库的第一个正式打标版本，目标是把 `S-UI Deployer` 从初始脚手架推进到一个可以实际管理单站点 S-UI 生命周期的工具版本。

## 本版本包含什么

### 1. 链路级 CLI

新增一组链路级命令，用于在已有站点内增量管理：

- 一个用户
- 一个入站
- 一个出站策略
- 一条路由绑定

已实现命令：

```bash
bin/sui-deploy chain-import-current <site.env>
bin/sui-deploy chain-list <site.env>
bin/sui-deploy chain-show <site.env> <chain-id>
bin/sui-deploy chain-plan-create <site.env> <chain.json>
bin/sui-deploy chain-apply-create <site.env> <chain.json>
bin/sui-deploy chain-plan-delete <site.env> <chain-id>
bin/sui-deploy chain-apply-delete <site.env> <chain-id>
```

当前支持：

- `direct`
- `shared`
- `dedicated`

### 2. 证书自动续签闭环

新增证书自动续签相关命令：

```bash
bin/sui-deploy cert-status <site.env>
bin/sui-deploy cert-renew <site.env> [--dry-run] [--force]
bin/sui-deploy cert-supervise <site.env>
bin/sui-deploy install-cert-supervisor <site.env>
```

闭环结构：

- **内环**：VPS 本机 `systemd timer`
- **外环**：工作站监督巡检

判据不依赖 `acme.sh` 输出文本，而是依赖：

- 证书文件存在
- 服务重启成功
- 对外 TLS 指纹与文件证书一致

### 3. 文档体系同步

这次版本补齐了当前能力对应的文档：

- README
- 新手手工搭建手册
- 工具辅助部署手册
- 安全与运维手册
- 自动化路线图
- 链路 CLI 路线图
- 证书自动续签控制闭环设计

## 已知边界

- 链路级 CLI 当前仍偏结构验证，不自动完成真实出口 IP 验证和协议级测速。
- 当前工具仍以单站点管理为主，不覆盖多站点批量编排。
- 证书自动续签已具备闭环，但真实长期运行表现仍需要继续积累运行样本。

## 升级建议

如果你已经在使用旧版工作目录：

1. 拉取 `v0.1.0`
2. 对照 `templates/config/env.example` 补齐证书自动续签字段
3. 对已有站点先运行：
   - `cert-status`
   - `install-cert-supervisor`
4. 如需增量管理新用户/入口，再使用链路级 CLI
